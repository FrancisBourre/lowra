Available nodes in context file

You will find below the various types of nodes possible inside the file defining the context ( applicationContext.xml ) for LowRA IoC.

  Nodes Context Description
  <application-loader> <bean> Defines the loader view used to display the assembler process (only one occurence).
  <argument> <%dynamic%>, <method-call> Creates a new argument for a function call or a class constructor.
  <bean> File root Required root node for the application context file (only one occurence).
  <default> Context Description
  <dll> <bean> Loads the specified *.swf file as a Dynamically Loaded Library (dll).
  <%dynamic%> <bean>, <root> The node name has no meaning in this case, it creates an new entity of the specified type and identified with the specified id.
  <item> <%dynamic% type="Dictionary"> Defines an entry in a Dictionary.
  <key> <item> Defines the key of the <item> parent note.
  <listen> <%dynamic%> Sets the <%dynamic%> parent node as listener of the specified id. The <listen> node is only for dynamic nodes which are outside of the <root> node.
  <method-call> <%dynamic%> Calls a method of the <%dynamic%> parent node.
  <property> <%dynamic%> Sets the initial value of the specified property of the <%dynamic%> parent node.
  <root> <bean> Defines the root node for the graphics structures (only one occurence). All childs of the <root> nodes will be considered as part of the display list.
  <value> <item> Defines the value of the <item> parent note.
